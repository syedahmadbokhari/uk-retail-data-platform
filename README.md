## рџ§± 1. ETL Pipeline

### Ingest
Copies all five source tables into `raw_*` staging tables. Acts as an immutable snapshot вЂ” source tables are never modified.

### Clean
Applies table-specific transformations:

| Table | Transformations |
|-------|----------------|
| **finance** | Drop null revenue, clip discount to `[0, 1]`, fill null prices with 0 |
| **brands** | Drop null brands, strip whitespace, title-case |
| **info** | Drop null product names, strip whitespace |
| **reviews** | Convert European decimal ratings (`"3,3"` в†’ `3.3`), clip to `[0, 5]` |
| **traffic** | Drop null/empty visit timestamps |

59 null/garbage rows removed в†’ **3,120 clean products**.

### Aggregate
Four analytics tables built from clean data:

- `analytics_brand_revenue` вЂ” brand totals, revenue share %
- `analytics_product_revenue` вЂ” product totals + revenue rank
- `analytics_monthly_traffic` вЂ” monthly visit counts
- `analytics_discount_impact` вЂ” revenue split by discount status

## вњ€пёЏ 2. Apache Airflow Orchestration

The pipeline is defined as a DAG in `pipeline/dags/retail_pipeline.py`, replacing the script runner with a proper workflow engine.

```
ingest_raw в†’ clean_tables в†’ build_analytics в†’ build_features в†’ build_similarity_matrix
```

Each step is a `PythonOperator` wrapping the existing ETL functions вЂ” no logic was duplicated.

DAG configuration:
- `schedule_interval = "@daily"`
- `catchup = False`
- `retries = 1`, `retry_delay = 5 minutes`
- Tagged: `retail`, `etl`, `ml`, `recommendations`

## рџ—„пёЏ 3. Database Layer вЂ” PostgreSQL + SQLite

`src/utils/db.py` uses **SQLAlchemy** with automatic database selection:

```python
# PostgreSQL вЂ” when DB_HOST is set
```

No code changes required in the ETL layer вЂ” the context manager pattern is identical regardless of which database is active. The `engine.begin()` transaction block handles commit/rollback automatically.
